# QVAC Transcription Whisper Addon v0.4.0 Release Notes

This is a major release with breaking changes. The `TranscriptionFfmpegAddon` module has been removed as part of the deprecation of `@qvac/util-transcription`.

## Breaking Changes

### Removed `TranscriptionFfmpegAddon`

The `transcription-ffmpeg.js` module and its TypeScript declarations have been removed. This module provided a combined audio decoding and transcription pipeline using `@qvac/util-transcription`.

**Removed files:**
- `transcription-ffmpeg.js`
- `transcription-ffmpeg.d.ts`
- `examples/example.ffmpeg.js`

### Removed `@qvac/util-transcription` Dependency

The `@qvac/util-transcription` package is now deprecated. This dependency has been removed from the package.

## Migration Guide

If you were using `TranscriptionFfmpegAddon`, please migrate to using `@qvac/sdk` which provides a unified transcription API with built-in audio decoding support.

### Before (v0.3.x)

```javascript
const TranscriptionFfmpegAddon = require('@qvac/transcription-whispercpp/transcription-ffmpeg')

const addon = new TranscriptionFfmpegAddon({
  loader,
  modelName: 'ggml-tiny.bin',
  params: { decoder: { streamIndex: 0 } }
}, whisperConfig)

await addon.load()
const response = await addon.run(audioStream)
```

### After (v0.4.0)

Use `@qvac/sdk` for transcription with audio decoding:

```javascript
const { Qvac } = require('@qvac/sdk')

const qvac = new Qvac({ /* config */ })
const transcription = await qvac.transcribe(audioStream)
```

## Other Changes

- Updated README.md to remove TranscriptionFfmpegAddon documentation
- Updated `test:dts` script to remove reference to deleted TypeScript declarations

## Upgrade Notes

- If you only use `TranscriptionWhispercpp` directly (the main export), no changes are required
- If you import from `transcription-ffmpeg`, you must migrate to `@qvac/sdk`
