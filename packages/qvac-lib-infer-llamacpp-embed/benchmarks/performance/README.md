# Performance Benchmarks

This directory contains performance benchmark runners for `@qvac/embed-llamacpp`.

## Setup

From `packages/qvac-lib-infer-llamacpp-embed`:

```bash
npm run performance:install
```

## Parameter Sweep

Run from `packages/qvac-lib-infer-llamacpp-embed`:

```bash
npm run benchmarks:param-sweep
```

This command runs:

1. `prepare-models.js --target addon`
2. Bare benchmark runner (`embed-parameter-sweep.js`) with static inputs from
   `benchmarks/performance/inputs.json`

## Addon Source Selection

Addon source is explicit (no automatic fallback). Use:

- `--addon-source local` (default): load local addon from `../../index`
- `--addon-source npm`: load installed `@qvac/embed-llamacpp`

Examples:

```bash
# Local addon (default)
npm run benchmarks:param-sweep -- --addon-source local

# npm package addon
npm run benchmarks:param-sweep -- --addon-source npm
```

## Optional Sweep Flags

- `--repeats 10`
- `--debug`

Default repeats per case:

- `5`

Default output directory and files:

- `benchmarks/performance/results/parameter-sweep/`
- `embed-parameter-sweep-<timestamp>.md`
- `embed-parameter-sweep-<timestamp>.jsonl`

Each case metric in outputs currently reports mean values across repeats (and single load/unload timings per case run).

## Runtime Defaults

Baseline/default runtime settings are defined in `embed-parameter-sweep.config.js`:

- `BENCH_DEFAULT_RUNTIME` (global defaults for all models)
- `MODEL_RUNTIME_OVERRIDES` (optional per-model overrides)

Model list and quantization files come from:

- `models.manifest.json`
- `resolved-models.json` (generated by model preparation)
